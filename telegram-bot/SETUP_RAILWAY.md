# Инструкция по настройке Telegram бота в Railway

## Шаг 1: Получение токена бота

1. Откройте Telegram и найдите бота [@BotFather](https://t.me/BotFather)
2. Отправьте команду `/newbot`
3. Следуйте инструкциям:
   - Введите имя бота (например: `ENTECH GROUP Bot`)
   - Введите username бота (должен заканчиваться на `bot`, например: `entechsite_bot`)
4. Скопируйте полученный токен (выглядит как: `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`)

## Шаг 2: Создание нового сервиса в Railway

1. Откройте ваш проект в Railway
2. Нажмите на кнопку **"New"** → **"GitHub Repo"** (или **"Empty Project"** если хотите загрузить код вручную)
3. Выберите репозиторий с вашим проектом
4. Railway автоматически определит все сервисы в проекте

## Шаг 3: Настройка сервиса Telegram бота

1. В списке сервисов найдите или создайте сервис для `telegram-bot`
2. Если сервис не определился автоматически:
   - Нажмите **"New"** → **"GitHub Repo"**
   - Выберите папку `telegram-bot` в репозитории
   - Или создайте **"Empty Service"** и подключите репозиторий

## Шаг 4: Настройка переменных окружения

1. Откройте настройки сервиса Telegram бота
2. Перейдите в раздел **"Variables"**
3. Добавьте переменную окружения:
   - **Key**: `BOT_TOKEN`
   - **Value**: токен, полученный от @BotFather

## Шаг 5: Настройка деплоя

Railway автоматически определит проект как Node.js приложение и выполнит:
- `npm install` - установка зависимостей
- `npm run build` - сборка TypeScript проекта
- `npm start` - запуск бота

Если автоматическое определение не сработало, убедитесь, что:
- В корне папки `telegram-bot` есть файл `package.json`
- В `package.json` указан скрипт `start`: `"start": "node dist/index.js"`

## Шаг 6: Проверка работы бота

1. После деплоя проверьте логи сервиса в Railway
2. Должно появиться сообщение: `🤖 Telegram бот запущен и готов к работе`
3. Откройте ваш бот в Telegram (по username, который вы указали при создании)
4. Отправьте команду `/start`
5. Вы должны получить приветственное сообщение

## Возможные проблемы

### Бот не отвечает
- Проверьте, что токен правильный в переменных окружения
- Проверьте логи сервиса на наличие ошибок
- Убедитесь, что сервис запущен (статус "Online")

### Ошибка при деплое
- Убедитесь, что все зависимости указаны в `package.json`
- Проверьте, что TypeScript компилируется без ошибок: `npm run build`
- Проверьте логи сборки в Railway

### Бот не видит команды
- Убедитесь, что бот запущен (проверьте логи)
- Попробуйте перезапустить сервис в Railway
- Проверьте, что используете правильного бота (правильный username)

## Дополнительная настройка

### Изменение приветственного сообщения

Отредактируйте файл `telegram-bot/src/index.ts`, измените текст в переменной `welcomeMessage`, затем:
1. Закоммитьте изменения
2. Railway автоматически задеплоит обновление

### Добавление новых команд

Добавьте обработчики команд в `telegram-bot/src/index.ts`:

```typescript
bot.onText(/\/команда/, async (msg) => {
  const chatId = msg.chat.id;
  await bot.sendMessage(chatId, 'Ответ на команду');
});
```
